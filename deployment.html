    2. Deployment

    Deployment

    This page lists deployment configuration. It is based on placeholders (variables) and tunable using an environment file or maven properties. You can check the pom for bundlebee-placeholder-import comment to see how to enable an externalised file to host the environment variables.

    Placeholders

    image.pullPolicy

    Deployment image pull policy (if image is moving - snapshot or latest, ensure to set it to Always). Default: IfNotPresent.

    hcms.version*

    Image tag version to deploy.
    hcms.resources.requests.cpu

    Kubernetes CPU requests. Default: 1.

    hcms.resources.memory

    Kubernetes resources memory to allocate. Default: 512Mi.

    image.registry*

    Base image registry to pull the image from.
    user.name

    Deployment user - injected in a label. Default: unknown.

    app.deploytime

    Label with the deployment time injected. Using maven setup it uses maven.build.timestamp property. Default: unset.

    hcms.replicas

    Number of replicas for the web application. Default: 1.

    bundlebee.environment

    Label hosting the environment name (can enable to identify deployment errors). Default: default.
